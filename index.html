Diograph.org

The Library for the Personal Web

Diograph.org is a Javascript library for the Personal Web developers. For the Personal Web itself, go to personal-web.org.

Getting started

  1. Create or get a Diograph document.
  2. Save document to a folder in a device, service or server.
  3. Open document with a Personal Browser.

Conceptual Model

Personal Web is a personal information system that sits between the WWW (global information system) and Files and Folders (local information system). Its purpose is to give people a better way to think and a neutral way to have digital information. The core of the Personal Web is the Diograph -data model that is used in the Personal Web applications and that is stored in any file system. The Diograph -data model is based on the Digital Habitat -conceptual model, the new way of thinking digital information. The design principle of the Personal Web is that our digital environment should be similar to our physical environment. The same principle is used also before in the desktop metaphor and paper/document metaphor Personal Web, respectively, consists of places like our physical environment. A place has owners, which are represented by avatars. A place again consists of rooms. A room is the main functional unit of the Personal Web. A room has walls, floor and sky as a ceiling. Each of these contain diories (digital memories), the basic information units of the Personal Web. Diories can be organized into a graph (semantic memory), a timeline (temporal memory), a map (spatial memory), a flowchart (procedural memory) or as a landscape of doors to other rooms. `place` -> `avatar`, `room` -> 'diory' (digital memory) 'room' -> `wall`, `floor`, `sky` -> `diory` (digital memory) (Conceptual model of the Personal Web)

Diory

Diory is the basic information unit and element in the Personal Web. A diory consists of name, text, image and address elements. Personal Web tools can attach buttons to diories.

Room is the container in the Personal Web. A room consists of walls, floor and sky as a ceiling. A room contains diories and tools.

Format

Diograph (digital biography) is a text document (JSON-LD format) for storing information of a room. 

room.json:

    {
        "diory": {...},
        "position": {...},
        "status": "on",
        "newThings": "2",
        "agents": [...],
        "doors": [...],
        "objects": [...],
        "moments": [...],
        "places": [...]
    }

Diory Format

Diory (digital memory) is the unit of information in the Personal Web. A diory relates to a room or a thing (agent, door, object, moment and place).

Thing Format

Doors, objects, agents, keys and tools are things. Things can be copied from a room to another. 

door.json:

    {
        "diory": "",
        "position": {...},
        "status": "on", // On, off
        "newThings": "", // New things since previous update
        "agents": [...],
        "key": ""
    }

agent.json:

    {
        "diory": "",
        "position": {},
        "status": "",
        "newThings": "",
        "tools": [],
        "keys": [
            {
                "ID": "", // Door
                "key": ""
            }
        ]
    }

object.json:

    {
        "diory": "",
        "position": {},
        "status": "",
        "newThings": "",
        "objects": []
    }

Dimension Format

Moment and place are dimensions. 

moment.json:

    {
        "diory": "",
        "position": {},
        "status": "",
        "newThings": "",
        "key": ""
        "objects": []
    }

place.json:

    {
        "diory": "",
        "position": {},
        "status": "",
        "newThings": "",
        "objects": []
    }

Security Model

The Personal Web is based on the Document Encryption security model. The room.json document is encrypted with the AES-256 algorithm. Agents have keys to the room or keys can be in doors in other rooms.

door.json:

    {
        "diory": "",
        "position": {...},
        "status": "on", // On, off
        "newThings": "", // New things since previous update
        "agents": [...],
        "key": ""
    }

agent.json:

    {
        "diory": "",
        "position": {},
        "status": "",
        "newThings": "",
        "tools": [],
        "keys": [
            {
                "ID": "", // Room
                "key": ""
            }
        ]
    }

Options to share a room:

  1. Send the room.json and key.
  2. Send door with key.
  3. Send door and agent with key.

Decrypt, encrypt with new key, send key to agents in the room.

Places (servers) may have the standard access control method. Is it possible to save with a token saved in the room?

Personal Browser

Personal Browser is a html5 application that reads Diograph JSON-LD format. Use here or download to here.


    http://personal-browser-domain-and-path?address&key
    http://personal-browser-domain-and-path?address&key&diory

Address is the URL-address of the room.json file. Key opens the door to the room, if it's locked.


    http://personal-browser-domain-and-path?room&diory
